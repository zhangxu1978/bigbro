# 剧情下一步推演重构计划

## 需求分析

### 问题
1. 当前"推演下一步"和"新剧情编织"都使用相同的`PLOT_DESIGNER_PROMPT`（剧情编织者）
2. 用户要求"推演下一步"应该使用不同的AI角色："命运织机"
3. 剧情步骤`plot_steps`表结构需要增加字段
4. 点击"推演下一步"需要弹出类似角色创建的模态框界面

### 目标
1. 分离"剧情编织者"和"命运织机"的prompt
2. 修改`plot_steps`表结构和相关代码
3. 创建"剧情步骤创建"模态框（类似角色创建界面）
4. 支持AI生成、AI重构功能

---

## 详细实现计划

### 1. 修改数据库表结构

**文件**: `utils/database.js` 和 `utils/dbOperations.js`

**plot_steps表新结构**:
```sql
CREATE TABLE IF NOT EXISTS plot_steps (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    plotId TEXT NOT NULL,
    stepNumber INTEGER NOT NULL,
    purpose TEXT NOT NULL,           -- 剧情设计目的
    emotion TEXT NOT NULL,            -- 调动观众情绪
    characters TEXT NOT NULL,         -- 涉及角色（JSON数组）
    summary TEXT NOT NULL,            -- 剧情简述
    obstacle TEXT,
    achievement TEXT,
    narrative TEXT,
    status TEXT DEFAULT 'pending',
    createdAt INTEGER DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (plotId) REFERENCES plots(id) ON DELETE CASCADE
)
```

**需修改的函数**:
- `createPlotStep()` - 添加新字段
- `getPlotSteps()` - 读取新字段
- `updatePlotStep()` - 更新新字段

---

### 2. 创建"命运织机"Prompt

**文件**: `public/life-simulator/plot-driver.js`

新增 `FATE_WEAVER_PROMPT` 常量，作为"推演下一步"的AI角色:
- 了解当前世界观和剧情规划
- 根据已规划的剧情步骤推演下一步
- 必须包含：剧情设计目的、调动情绪、涉及角色、剧情简述

---

### 3. 创建剧情步骤模态框

**文件**: `public/life-simulator.html`

新增模态框 `step-creator-modal`，包含字段:
- 剧情设计目的（必填）
- 调动观众情绪（必填）
- 涉及角色（必填）
- 剧情简述（必填）
- 阻碍（可选）
- 达成（可选）
- 剧情描述（可选）

按钮: AI生成、重新生成、保存

---

### 4. 修改前端逻辑

**文件**: `public/life-simulator/plot-driver.js`

**新增函数**:
- `showStepCreator()` - 显示步骤创建模态框
- `generateStep()` - AI生成步骤（使用命运织机prompt）
- `regenerateStep()` - AI重构步骤（类似角色重新生成，保留已填字段）
- `saveStep()` - 保存步骤

**修改函数**:
- `generateNextStep()` - 改为弹出模态框而非直接调用AI
- `renderPlotSteps()` - 适配新字段显示
- `savePlotStep()` - 适配新字段结构

---

### 5. 修改API接口

**文件**: `api/plotdriver.js` 或相关API文件

确保`/plot-steps`相关接口支持新字段的CRUD操作。

---

## 实施步骤

1. **Step 1**: 修改数据库表结构（database.js + dbOperations.js）
2. **Step 2**: 创建命运织机Prompt（plot-driver.js）
3. **Step 3**: 创建剧情步骤模态框HTML（life-simulator.html）
4. **Step 4**: 添加模态框CSS样式（life-simulator.css）
5. **Step 5**: 实现步骤创建/生成/重构逻辑（plot-driver.js）
6. **Step 6**: 修改generateNextStep和renderPlotSteps适配新结构
7. **Step 7**: 测试验证

---

## 注意事项

- 保持与现有角色创建界面风格一致
- AI生成时需要提供完整的上下文（世界观、剧情规划、已有步骤）
- 重新生成时保留用户已填写的内容，只填充空缺字段
- 所有步骤必须包含"剧情设计目的"
